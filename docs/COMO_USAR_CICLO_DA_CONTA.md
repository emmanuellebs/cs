# 📈 Como Usar o Ciclo da Conta

Explicação detalhada sobre o campo "Ciclo da Conta" e como diferencia do Board.

---

## **O que é Ciclo da Conta?**

**Ciclo da Conta** é um **campo auxiliar** que marca o **estágio estratégico** do cliente.

É diferente do Board:
- **Board** = etapa operacional (o que você está FAZENDO hoje)
- **Ciclo da Conta** = saúde estratégica (em que ETAPA DE RELACIONAMENTO está)

---

## **Por que Existem Dois?**

### **Exemplo:**
```
ACME Corp

BOARD:              CICLO DA CONTA:
Acompanhamento 2    Engajamento
(etapa operacional) (estágio estratégico)

Significado:        Significado:
"Hoje estou em      "Estrategicamente,
acompanhamento      cliente está muito
contínuo."          engajado e satisfeito."
```

Ambas são verdadeiras, mas respondem perguntas diferentes.

## **Os 10 Estágios**

### **1️⃣ ONBOARDING**

**O que é:**
Cliente novo, começando adoção inicial.

**Características:**
- Contrato tem menos de 90 dias
- Cliente fazendo primeiros passos
- Usando plataforma pela primeira vez
- Equipe interna em treinamento

**Quando marcar:**
- Novo cliente assinou
- Até 90 dias após go-live

**Quando sair:**
- Passou 90 dias
- Cliente usando regularmente
- Adoção atingiu x% de usuários

**Ações esperadas:**
- Reuniões semanais
- Suporte intensivo
- Treinamento

**Exemplo:**
```
ACME assinou em 01/01.
Marcado como Onboarding em 01/01.
Go-live em 01/03.
Permanece Onboarding até 30/03 (90 dias).
Muda para Ativo em 01/04.
```

---

### **2️⃣ ATIVO**

**O que é:**
Cliente saiu do onboarding, está usando regularmente a plataforma.

**Características:**
- Usando 60%+ de funcionalidades
- Passaram 90 dias do onboarding
- Adoção consolidada
- Sem grandes problemas

**Quando marcar:**
- Onboarding completo
- Uso regular documentado

**Quando sair:**
- Cliente entrando em estrutura de adoção organizada (passa para Adoção)
- OU cliente mudando para próximo estágio

**Ações esperadas:**
- Reuniões mensais
- Monitorar Health Score
- Observar engajamento

**Exemplo:**
```
ACME finaliza onboarding em 30/03.
Move para Ativo em 01/04.
Fica em Ativo enquanto estável e usando bem.
```

---

### **3️⃣ ADOÇÃO**

**O que é:**
Cliente está em programa estruturado de adoção (treinamento avançado, change management, departamentos).

**Características:**
- Programa formal de adoção em andamento
- Novos departamentos entrando
- Treinamento avançado
- Customizações sendo feitas

**Quando marcar:**
- Cliente começou novo programa de adoção
- Há estrutura de acompanhamento definida

**Quando sair:**
- Programa concluído
- Cliente move para Engajamento (naturalmente engajado)

**Ações esperadas:**
- Reuniões bi-semanais
- Plano de Sucesso ativo
- Treinamento específico

**Exemplo:**
```
ACME em Ativo por 2 meses.
Em 01/06, decidem expandir para 3 novos departamentos.
Marcado como Adoção em 01/06.
Programa de 60 dias (até 31/07).
Move para Engajamento em 01/08.
```

---

### **4️⃣ ENGAJAMENTO**

**O what é:**
Cliente muito engajado com a plataforma, equipe investindo tempo, participando de reuniões, dando feedback.

**Características:**
- Alto engajamento
- Feedback positivo regular
- Participando de reuniões/eventos
- Tomando decisões com base em dados da plataforma

**Quando marcar:**
- Cliente demonstrando engajamento consistente
- Participação ativa em reuniões

**Quando sair:**
- Cliente move para Advocacy (promoter)
- OU fica estável sem evolução (volta para Ativo)

**Ações esperadas:**
- Reuniões mensais com executivos
- Coleta de feedback estruturada
- Análise de casos de sucesso

**Exemplo:**
```
ACME em Adoção até 31/07.
Em 01/08, move para Engajamento.
Cliente dando feedback regular.
Participando de webinars.
Health Score: 85.
```

---

### **5️⃣ EXPANSÃO**

**O que é:**
Cliente crescendo: novos usuários, novos módulos, novos departamentos, aumento de receita.

**Características:**
- MRR aumentando
- Novos casos de uso
- Expansão vertical ou horizontal
- Cliente investindo mais

**Quando marcar:**
- Expansão aprovada
- Setup técnico começando
- Novos usuários sendo adicionados

**Quando sair:**
- Expansão concluída
- Cliente volta para Engajamento (ou Ativo, depende)

**Ações esperadas:**
- Reuniões de alinhamento
- Treinamento de novos usuários
- Monitorar adoção dos novos módulos

**Exemplo:**
```
ACME em Engajamento.
Em 15/03, cliente quer adicionar 200 usuários (3 depto).
Marcado como Expansão em 15/03.
Expansão concluída em 30/04.
Volta para Engajamento em 01/05.
```

---

### **6️⃣ ADVOCACY**

**O que é:**
Cliente é promoter ativo: faz indicações, participa de eventos, permite case study, comunidade.

**Características:**
- NPS alto (promoter)
- Fazendo indicações
- Participando de eventos
- Permitindo referências
- Possível parceria

**Quando marcar:**
- Cliente demonstrou comportamento de promoter
- Indicou clientes / está disposto a fazer referência

**Quando sair:**
- Cliente move para Renovação (se vencimento próximo)
- OU volta para Engajamento (se reduz atividade de advocacy)

**Ações esperadas:**
- Reuniões estratégicas
- Programas de advocacy (caso study, event speaking, etc)
- Oportunidades de parcerias

**Exemplo:**
```
ACME NPS: 42 (promoter).
Indicou 2 clientes em 2 meses.
Disposto a fazer case study.
Marcado como Advocacy em 01/05.
```

---

### **7️⃣ RENOVAÇÃO**

**O que é:**
Contrato próximo de vencer, cliente em processo de renovação.

**Características:**
- Contrato vence em menos de 90 dias
- Negociação em andamento
- Proposta de renovação em andamento

**Quando marcar:**
- 90 dias antes do vencimento

**Quando sair:**
- Renovado → volta para Ativo/Engajamento/Advocacy
- OU Churn → vai para Churn

**Ações esperadas:**
- Reunião de revisão
- Proposta comercial
- Negociação com cliente
- Assinatura de contrato

**Exemplo:**
```
ACME contrato vence 30/06.
Em 01/04, marcado como Renovação (90 dias).
Reúne com cliente, cria proposta, negocia.
Em 30/05, cliente renova.
Move para Renovado em 01/06.
```

---

### **8️⃣ RENOVADO**

**O que é:**
Cliente renovarou o contrato com sucesso.

**Características:**
- Contrato foi renovado
- Novo período começou ou está começando
- Cliente continua como cliente

**Quando marcar:**
- Contrato renovado (assinado e novo período iniciado)

**Quando sair:**
- Move para Engajamento (volta ao fluxo normal)
- Ou mantém Renovado até novo período começar

**Ações esperadas:**
- Celebrar com cliente
- Revisar o que funcionou bem
- Planejar próximos 12 meses

**Exemplo:**
```
ACME renova contrato em 30/05.
Marcado como Renovado em 01/06.
Em 15/06, move para Engajamento (novo período iniciado).
```

---

### **9️⃣ RISCO**

**O que é:**
Cliente em risco de churn. Detectado problema, desengajamento, que pode virar cancelamento.

**Características:**
- Health Score caindo
- Sem contato por 30+ dias
- Reclamações
- Concorrente oferecendo alternativa
- Uso caindo

**Quando marcar:**
- IMEDIATAMENTE quando risco é identificado
- Pode sobrepor outro estágio (ex: cliente está em Engajamento, mas com Risco identificado)

**Quando sair:**
- Risco mitigado → volta para estágio anterior
- OU cliente churn → move para Churn

**Ações esperadas:**
- Reunião executiva urgente
- Plano de remediação
- Acompanhamento semanal

**Exemplo:**
```
ACME estava em Engajamento.
Health Score caiu de 85 para 60.
Sem contato há 45 dias.
Marcado como Risco em 08/03.
CSM agenda reunião urgente.
Se melhora, volta para Engajamento.
Se não, vai para Churn.
```

---

### **🔟 CHURN**

**O que é:**
Cliente não está mais cliente. Cancelou contrato.

**Características:**
- Contrato cancelado ou não renovado
- Cliente não é mais usuário ativo
- Relacionamento encerrado

**Quando marcar:**
- Cliente comunicou cancelamento
- Contrato venceu sem renovação
- Cliente virou ex-cliente

**Quando sair:**
- Nunca (cliente removido do acompanhamento)

**Ações esperadas:**
- Documentar razão do churn
- Reunião de exit (se possível)
- Análise de aprendizados

**Exemplo:**
```
ACME estava em Renovação.
Em 15/06, cliente disse: "Vamos para concorrente, oferta melhor."
Marcado como Churn em 15/06.
Data de fim: 30/06.
Documento criado com lições aprendidas.
```

## **Como Atualizar Ciclo da Conta**

### **Onde fica o campo:**

1. Abra um Cliente no Jira
2. Procure pelo campo **"Ciclo da Conta"** (lado direito)
3. Clique no campo
4. Escolha o novo estágio
5. Clique **Salvar** (ou salva automaticamente)

### **Quando atualizar:**

- ✅ Quando há mudança REAL de estágio
- ✅ Semanal (revise se algo mudou)
- ✅ Após reunião importante
- ✅ Quando detectar risco
- ✅ Quando cliente renova

### **Quando NÃO atualizar:**

- ❌ Não precisa ser diariamente
- ❌ Não altere só porque quer "limpar"
- ❌ Altere quando há motivo real

## **Ciclo da Conta vs Board**

### **DIFERENÇAS CLARAS:**

| Aspecto | Ciclo da Conta | Board |
|---------|---------------|-------|
| **O quê** | Estratégia | Operação |
| **Quantos estágios** | 10 | 8 |
| **Frequência de atualização** | Semanal (ou quando muda) | Conforme avança |
| **Propósito** | Entender saúde | Ver progresso |
| **Exemplo** | Cliente marcado como Risco | Cliente em coluna Renovação |
| **Quem vê** | Executivos, métricas | Time do dia a dia |

### **EXEMPLO PRÁTICO:**

```
ACME Corp

DIA 1 (Cliente novo):
  Board: Análise de Perfil
  Ciclo: Onboarding
  Significado: Cliente novo, começando

DIA 45 (Meio do onboarding):
  Board: Implantação
  Ciclo: Onboarding
  Significado: Dentro do plano

DIA 90 (Go-live):
  Board: Lançamento
  Ciclo: Onboarding
  Significado: Cliente lançado

DIA 120 (Pós-lançamento):
  Board: Acompanhamento 1
  Ciclo: Ativo
  Significado: Cliente usando bem, saiu do onboarding

DIA 180 (Estável):
  Board: Acompanhamento 2
  Ciclo: Engajamento
  Significado: Cliente engajado, estável, bom relacionamento

DIA 200 (Descoberto problema):
  Board: Acompanhamento 2 (ainda)
  Ciclo: Risco
  Significado: Operacionalmente em Acompanhamento, mas estrategicamente em risco

DIA 210 (Risco mitigado):
  Board: Acompanhamento 2 (continua)
  Ciclo: Engajamento (volta)
  Significado: Risco passou, cliente engajado novamente

DIA 250 (Expansão aprovada):
  Board: Expansão
  Ciclo: Expansão
  Significado: Cliente expandindo com sucesso

DIA 300 (Pós-expansão):
  Board: Acompanhamento 2
  Ciclo: Advocacy
  Significado: Cliente estável, promoter, sem novos projetos

DIA 350 (90 dias para renovação):
  Board: Renovação
  Ciclo: Renovação
  Significado: Em processo de renovação

DIA 365 (Renovou!):
  Board: Acompanhamento 2 (volta)
  Ciclo: Renovado
  Significado: Renovação bem-sucedida, novo período começou
```

## **Como Usar Ciclo para Tomar Decisões**

### **Clientes em RISCO:**
- Filtrar Ciclo = Risco
- Revisar semanalmente
- Agendar reuniões urgentes
- Objetivo: Mitigar ou documentar churn

### **Clientes em ADVOCACY:**
- Filtrar Ciclo = Advocacy
- Oferecer programas especiais
- Pedir indicações
- Fazer case studies

### **Clientes em RENOVAÇÃO:**
- Filtrar Ciclo = Renovação
- Priorizar negociações
- Garantir propostas no prazo
- Comemorar sucesso

### **Para Relatórios:**
```
Dashboard Executive:
- Total clientes: 50
- Onboarding: 3 (6%)
- Ativo: 10 (20%)
- Engajamento: 20 (40%)
- Advocacy: 10 (20%)
- Risco: 5 (10%)
- Churn: 2 (4%)

Ações: Focar em 5 em risco.
```
